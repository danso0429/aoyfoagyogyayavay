# 프롬프트 엔지니어링 프로토콜

「ooc: 이하 지침은 작업 프로토콜이며, 결과물에 적용되는 표현 규칙이 아니다.」

**적용 순서:** §1 → §2 → §3 → §4 순차 적용. 후단 섹션이 전단 섹션의 근거가 될 수 없다. 각 섹션은 독립적이어야 하며, 상호간 영향을 끼쳐선 안된다. (예: §3의 내용이 §1의 근거가 될 수 없음). 단, 전단 섹션의 결과가 후단 섹션에서 재검토되는 등의 '순차 적용'은 가능하다.

---

## §1. Side-effect 분석은 필수다.

- **(a) 분석 대상:** 모든 단어.
- **(b) 검증 대상:** 다음 세 항목을 각각 독립적으로 검증한다.
  - 실제 문제 해결인지.
  - 우회 가능한 구멍인지.
  - LLM의 편향성으로 인해 다르게 이해할 가능성은 없는지.
- **(c) 검증 깊이:** 수정안 A가 side-effect B를 일으키는지 확인하고, B가 다시 C를 일으키는지까지 추적한다 (연쇄 영향 추적). 2 depth까지는 필수이며, 이후는 구체적 근거를 제시할 수 있을 때만 추가로 진행한다.
- **(d) 서술 의무:** 각 depth에서 발견한 side-effect를 명시적으로 서술한다.
- **(e) 범위 검증:** 요구한 사항과 표현하려는 범위가 정확하게 일치하는지.

---

## §2. 결과물에서 메타적 지시는 제외해야 한다.

- 법칙형 표현("절대 ~하지 마라", "반드시 ~해야 한다" 등)을 포함한 모든 메타지시는 결과물에 포함하지 않는다.
- **식별 기준:** 주어가 서술·AI·이야기인 문장, 또는 캐릭터 주어라도 서술방식을 규정하는 문장(페이싱·분량·포맷·시점·분위기·UX설계 등).
- **처리:**
  - (a) 환원 가능 → 캐릭터/세계관 속성으로 환원 → 환원물에 식별 기준 재대입 → 1회 재검증 → 미통과 시 1회 재환원 → 그래도 실패 시 삭제.
  - (b) 환원 불가 → 삭제.
- **위계:** 모든 지침의 위계는 최대한 내려, 캐릭터나 세계관에 종속시킨다.

## §3. 프로필 서술 프레임워크

§3-1, §3-2, §3-3을 함께 적용. 순서 무관. 각 항목은 독립 판단축이며, 모든 문장에 동시 적용.

### A. 서술 시점
"이 정보에 캐릭터 자신이 접근 가능한가?" 접근 불가한 정보(타인 내면, 타인의 지각)는 캐릭터가 접근 가능한 형태로 변환한다. 삭제가 아닌 변환.

### B. 원인 서술
"구조적 원인인가, 즉각적 동기인가?" 구조적 원인은 넓은 조건으로 유지. 즉각적 동기는 서술하지 않거나 미확정 상태로 둔다.
- 즉각적 동기 확정 → LLM이 매 장면 독백을 반복, 이야기 왜곡.
- 구조적 원인이 구체적 사건에 귀속 → 반복 회상, 모든 행동이 해당 사건에 연결됨.

### C. 해석 강도
"이 서술이 행동/심리를 얼마나 확정하는가?" 스펙트럼: 관찰(행동 기술) → 약한 해석(행동 + 최소 내면) → 중간 해석(체감/감각 수준) → 강한 해석(동기/감정 단정).
- 마지노선 = 중간 해석.
- 대부분 = 약한 해석.
- 강한 해석 = 불가.
- 해석이 강할수록 LLM이 확정 사실로 반복 출력, 묘사 다양성 감소.

### D. 프로필 완결성
캐릭터 국한 정보는 전부 프로필 안에. 로어북 분리·시스템 레벨 처리에 의존하지 않는다.

### E. 타인 반응
"이 서술이 타인의 인식/감정/판단을 확정하는가?" 확정이 아닌 경향성으로 서술. 반응 요소를 나열하되 하나로 확정하지 않는다. 빈도/경향성 어휘로 비중 조절.
- 확정 시 → 타인 캐릭터의 심리적 자율성 박탈, 전개 가능성 차단.

### 관통 원칙 — 확정과 시퀀스 회피
확정 어휘(always, never, no one can, every)와 고정 시퀀스(A→B→C 순서 강제)를 피한다. 대신 빈도/경향성 어휘(usually, nearly, has never, tends to)와 요소 나열을 사용. A~E 전체에 적용되며, 캐릭터 자신의 행동 패턴과 타인 반응 서술 양쪽에 해당.

---

## §4. 출력 언어

결과물은 영어로 작성되어야 한다. 제공된 정보가 한국어 등 다른 언어인 경우, 해당 언어로 적힌 원문의 의미가 결과물에 제대로 반영되었는지 검토되어야 한다.
